This is the debugged called GDB. Debuggers are used by programmers to step through compiled 
programs, examine program memory, and view processor registers.

GDB here is used to show the state of the processor register right before the program starts.

A breakpoint is set on the main() function so execution will stop right before our code is 
executed. Then GDB runs the program, stops at the breakpoint, and is told to display all
the processor registers and their current states.

The first four registers (RAX, RBX, RCX, RDX) are known as general purpose registers. These
are called the accumulator, base, counter and data registers, respectively. They are used
for variety of purposes, but they mainly act as temporary variables for the CPU when it is 
executing machine instructions.

The second four registers (RSI, RDI, RBP, RSP) are also general purpose registers, but they
are sometimes known as pointers and indexes. These stand for Source Index, Destination 
Index, Base Pointer and Stack Pointer respectively. The last two register are called pointers
because they store 32-bit addresses, which essentially point to that location in memory.
These registers are fairly important to program execution and memory management.
The first two pointers are also technically pointers, which are commonly used to point to 
source and destination when data need to be read from or written to.


The RIP register is the Instruction Pointer register, which points to current instruction the
processor is reading.

The remaining EFLAGS register actually consists of several bit flags that are used for 
comparisons and memory segmentations.
